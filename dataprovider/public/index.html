    Intro

    Twapperlyzer is divided in two parts one is responseble for download and aggregating data and the other one is presenting it. This is the API for the first part

    Functions

    All Functions can be called with a callback parameter, in case you want to use JSONP.

    getArchiveList

    You call the function here

    This function download the archive list messages of a given your twapperkeeper instance and send it back to the user.

    Mandatory Parameter

    • param: String ytkUrl The url auf a your twapperkeeper instance

    Optional Parameter

    • param: Function callback The callback function in case you want to use JSONP

    analyseArchive

    You call the function here

    This function download messages from a given your tawpper keeper instance an save the basic information in the database and start the different analyse functions. These functions will update the database document when the finished. Note it will not update a existing archive analyses use updateArchive

    Mandatory Parameter

    • param: String ytkUrl The url auf a your twapperkeeper instance

    • param: Number l The limit of messages that should be analysed

    • param: Number id The id of the archive.

    Optional Parameter

    • param: String nort With on passed the messages staring with RT will be will filtered

    • param: String startDate The date from which on messages get analysed. The format is YYYY-MM-DD

    • param: String endDate The date till which on messages get analysed. The format is YYYY-MM-DD

    • param: String from_user Only the messages of that user will be analysed

    • param: String text Only the messages containing this string will be analysed

    • param: String o The Order in which the messages will be analysed. Valid parameters are either a (ascending)or d (descending)

    • param: String lang The language filter. Valid parameter are to letter codes. like en,fr etc.

    • param: String dbHost The host of a remote CouchDB instance

    • param: String dbname The name of a remote CouchDB database

    • param: Number dbPort The port of a remote CouchDB instance, if not given port 5984 will be used

    • param: String dbUser The name of a user for the remote CouchDB database

    • param: String dbPw The password of a user for the remote CouchDB database

    • param: String dbSecure If the CouchDb is reachble over hhtps this has to set to true

    • param: Function callback The callback function in case you want to use JSONP

    updateArchive

    You call the function here

    This function update a existing CouchDB document it download new messages and start the different analyse functions. These functions will update the database document when the finished.

    Mandatory Parameter

    • param: String docID The id of the CouchDB document that should be updated

    • param: Number l The limit of messages that should be analysed

    Optional Parameter

    • param: String dbHost The host of a remote CouchDB instance

    • param: String dbname The name of a remote CouchDB database

    • param: Number dbPort The port of a remote CouchDB instance, if not given port 5984 will be used

    • param: String dbUser The name of a user for the remote CouchDB database

    • param: String dbPw The password of a user for the remote CouchDB database

    • param: String dbSecure If the CouchDb is reachble over hhtps this has to set to true

    • param: Function callback The callback function in case you want to use JSONP
